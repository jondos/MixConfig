Using the Mix-Software as 'command line JAP'

The mix software can be used as a replacement for JAP. You can download a precompiled binary for linux x86 or you have to get the sources and compile it on your own. Either download the nightly generated Source Code Tarball or use anonymous CVS to get the newest version.

Use:
cvs -d :pserver:anonymous@cvs.inf.tu-dresden.de:/home/sk13/cvssource co proxytest
Now you need to compile it:
./configure

make
You should get a executable file called 'mix'. Do a quick test to see if it starts:
./mix -?
This should list a short description of the command line options. Before using the software as a local proxy you have to decide which mix-cascade to use. You can use JAP to get the addresses of all available mixes. The address of the 'Dresden-Dresden'-cascade for instance is:
mix.inf.tu-dresden.de:6544
Now you are ready to start the program. The parameters are:
-j --> to tell the mix to act as local proxy (like JAP)

-p [IP:]Port --> local port to use (and IP if given)

-n mix.inf.tu-dresden.de:6544 --> to which cascade to connect ('Dresden-Dresden' in this example)

optional parameters:
-l --> Directory where logfile should be placed (if omitted syslog is used)

-d --> to tell the mix to be a daemon (run in background)

If you have started the local proxy you should get messages like:
[2001/05/03-11:54:26, info ] Anon proxy started!
[2001/05/03-11:54:26, info ] Using: OpenSSL 0.9.6 24 Sep 2000!
[2001/05/03-11:54:26, info ] Starting MIX...
[2001/05/03-11:54:26, info ] Try connecting to next Mix...
[2001/05/03-11:54:26, info ] connected!
[2001/05/03-11:54:26, info ] Chain-Length: 2
